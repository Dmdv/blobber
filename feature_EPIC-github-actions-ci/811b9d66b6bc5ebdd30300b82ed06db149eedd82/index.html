1. Create/Restore existing wallet
Request Description
A wallet is registered or retrieved by sending a request to all miners on the network. **Requirements to make this request:** **[BIP39](https://en.bitcoinwiki.org/wiki/Mnemonic_phrase) Mnemonic**: A 24 word representation of the wallet private key seed with a passphrase of "0chain-client-split-key". There are [utilities](https://iancoleman.io/bip39/) that can help you generate this. **BLS BN254 private key:** Thee private key generated from the BIP39 mnemonic seed. 0Chain Keys are [BLS](https://en.wikipedia.org/wiki/BLS_digital_signature) with a [BN254 curve](https://neuromancer.sk/std/bn/bn254) . There are [many](https://github.com/herumi/bls) / [libraries](https://github.com/herumi/bls-wasm) to help you generate this key. **BLS BN254 public key hex string**: The public key calculated from the previously generated private key, formatted as a hex string. **Client/wallet id**: A SHA3-256 hash of the public key **hex dump** - for an example of how this hex dump can be generated, see the pre-request script of this request in postman. A consensus of 20% is required from the miners, after which you can be sure that your request will be propagated. The example below shows a single miner call but assumes the call will be made to all miners retrieved in step 1
